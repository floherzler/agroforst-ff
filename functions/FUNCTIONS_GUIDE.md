# Functions Guide

This repository now uses one standard for Appwrite functions:

- runtime: `node-22`
- entrypoint: `src/main.js`
- build command: `npm install --omit=dev`
- module format: ESM (`"type": "module"`)

Use this guide when you add or update any function.

Function IDs should be stable, human-readable IDs that match the function
name whenever possible.

Examples:

- `createMembership`
- `createOrder`
- `verifyPayment`

Avoid autogenerated IDs for long-lived functions. In this repository,
`createOrder` should use the Appwrite function ID `createOrder`, not a random
generated ID.

## Folder Template

Create a new folder under [`/home/flo178/projects/agroforst-ff/functions`](/home/flo178/projects/agroforst-ff/functions) with this shape:

```text
functions/<functionName>/
  README.md
  package.json
  src/
    main.js
```

Minimal `package.json`:

```json
{
  "name": "agroforst-function-<function-name>",
  "private": true,
  "type": "module",
  "dependencies": {
    "node-appwrite": "22.1.3"
  }
}
```

If the function does not call Appwrite APIs, only add the packages it actually
needs.

## Handler Template

Use this as the default starter:

```js
import { Client } from "node-appwrite";

function ok(res, data, status = 200) {
    return res.json(data, status);
}

function fail(res, msg, status = 400, extra = {}) {
    return res.json({ success: false, error: msg, ...extra }, status);
}

function readEnv(...keys) {
    for (const key of keys) {
        const value = process.env[key];
        if (typeof value === "string" && value.length > 0) {
            return value;
        }
    }
    return "";
}

function readHeader(req, key) {
    const headers = req?.headers ?? {};
    if (typeof headers.get === "function") {
        return headers.get(key) ?? headers.get(key.toLowerCase()) ?? "";
    }
    return headers[key] ?? headers[key.toLowerCase()] ?? headers[key.toUpperCase()] ?? "";
}

async function extractBody(req) {
    try {
        const body = await req.json();
        if (body && typeof body === "object") {
            return body;
        }
    } catch {
        // ignore
    }

    for (const candidate of [req?.bodyJson, req?.bodyText, req?.bodyRaw, req?.payload]) {
        if (typeof candidate === "string" && candidate.length > 0) {
            try {
                return JSON.parse(candidate);
            } catch {
                continue;
            }
        }
        if (candidate && typeof candidate === "object") {
            return candidate;
        }
    }

    return {};
}

export default async ({ req, res, log, error }) => {
    try {
        const callerId = readHeader(req, "x-appwrite-user-id");
        const endpoint = readEnv("APPWRITE_FUNCTION_API_ENDPOINT");
        const projectId = readEnv("APPWRITE_FUNCTION_PROJECT_ID");
        const apiKey = readEnv("APPWRITE_FUNCTION_API_KEY", "APPWRITE_API_KEY") || readHeader(req, "x-appwrite-key");

        if (!endpoint || !projectId || !apiKey) {
            return fail(res, "Function endpoint, project ID, or API key is not configured", 500);
        }

        const body = await extractBody(req);

        const client = new Client()
            .setEndpoint(endpoint)
            .setProject(projectId)
            .setKey(apiKey);

        log(`Caller present: ${Boolean(callerId)}`);

        return ok(res, { success: true, body });
    } catch (caughtError) {
        const msg = String(caughtError?.message ?? caughtError ?? "Unknown error");
        error(msg);
        return fail(res, "Internal error", 500, { details: msg });
    }
};
```

## Rules For This Repo

- Do not use `Deno.env`, `npm:` imports, `jsr:` imports, or `https://deno.land/...`.
- Read env only from `process.env`.
- Prefer plain helpers inside each function unless repetition becomes large.
- Keep response helpers consistent: `ok(...)` and `fail(...)`.
- Accept Appwrite request body variants:
  - `await req.json()`
  - `req.bodyJson`
  - `req.bodyText`
  - `req.bodyRaw`
  - `req.payload`
- If the function needs Appwrite auth, read:
  - `x-appwrite-user-id`
  - `x-appwrite-key`
- Keep logs short and searchable. Prefix them with the function name.

## How To Register A New Function

### 1. Add it to the local manifest

Update [`/home/flo178/projects/agroforst-ff/functions/local-manifest.json`](/home/flo178/projects/agroforst-ff/functions/local-manifest.json):

```json
"myFunction": {
  "source": "functions/myFunction",
  "runtime": "node-22",
  "entrypoint": "src/main.js",
  "commands": "npm install --omit=dev"
}
```

If the remote Appwrite function name differs from the repo folder, the manifest
key must match the remote Appwrite function name.

Example already used in this repo:

- remote function: `createOrder`
- repo source folder: [`/home/flo178/projects/agroforst-ff/functions/createOrder`](/home/flo178/projects/agroforst-ff/functions/createOrder)

When you create a new Appwrite function, prefer:

```bash
appwrite functions create --function-id createOrder --name createOrder ...
```

not a generated ID.

### 2. Add local env defaults

Update [`/home/flo178/projects/agroforst-ff/scripts/run-function-local.sh`](/home/flo178/projects/agroforst-ff/scripts/run-function-local.sh)
inside `emit_defaults()` so local runs get the env values they need.

If the function needs manual overrides in `.env`, also add a new section to
[`/home/flo178/projects/agroforst-ff/.env.example`](/home/flo178/projects/agroforst-ff/.env.example):

```env
# [myFunction]
MY_FUNCTION_SETTING=
```

### 3. Document it

Add a short [`README.md`](/home/flo178/projects/agroforst-ff/functions) in the
function folder with:

- purpose
- required env vars
- example payload
- auth assumptions

## Local Workflow

Use:

```bash
scripts/appwrite-local-dev.sh <function-id>
```

What this does:

1. pulls the current Appwrite functions into `.appwrite-local`
2. overlays them with the repo-owned Node sources
3. rewrites local Appwrite config to `node-22`
4. builds and runs the function with Docker via the Appwrite CLI

Examples:

```bash
scripts/appwrite-local-dev.sh createMembership
scripts/appwrite-local-dev.sh createOrder
scripts/appwrite-local-dev.sh verifyPayment
```

If you changed remote metadata and want a clean pull first:

```bash
APPWRITE_LOCAL_REFRESH=1 scripts/appwrite-local-dev.sh createMembership
```

## Deployment Workflow

The safe order is:

1. update repo code under `functions/`
2. update [`/home/flo178/projects/agroforst-ff/functions/local-manifest.json`](/home/flo178/projects/agroforst-ff/functions/local-manifest.json)
3. verify locally with `scripts/appwrite-local-dev.sh`
4. from `.appwrite-local`, update runtime metadata if needed
5. push functions with the Appwrite CLI

Useful commands:

```bash
appwrite functions list
appwrite functions update --function-id <id> --name <name> --runtime node-22 --entrypoint src/main.js --commands "npm install --omit=dev"
cd .appwrite-local
appwrite push functions --with-variables
```

## Review Checklist

Before merging function work, check:

- function uses `src/main.js`
- `package.json` exists and only includes needed deps
- no Deno-only APIs remain
- env names are documented
- local run succeeds
- payload validation returns clear `400` errors
- auth checks return `401` or `403` clearly
- logs do not leak secrets
